# FOSE 2026 SAGA 論文 要件整理

## 0. この文書の役割

この文書では、FOSE 2026 へ向けた SAGA 論文の要件を整理する。プロジェクト全体のビジョン、
AAT が目指すもの、その中で SAGA が達成したことを固定し、論文の主張と構成を判断するための
共通基準を作る。

最初に、次の三層を順に言語化する。論文のページ配分や投稿作業は、この三層から導く。

1. このプロジェクトが実現したい大きなビジョン
2. その中で AAT が担う理論的な役割
3. AAT の研究計画における SAGA の位置

数学的な主張の正本は AAT 数学本文、証明状態の正本は Lean 実装、tooling 仕様の正本は
ArchSig の実装と現行 tooling docs に置く。本書はそれらを参照し、論文のための読みを整理する。

## 1. 大きなビジョン

### 1.1 出発点となる構想

このプロジェクトの出発点は、次の構想である。

> **ソフトウェア工学の再設計――ソフトウェア進化を計算可能にする。**

ソフトウェア工学は、変化し続ける software を扱う学問である。設計、実装、review、CI、運用、AI による
変更生成を、software evolution を構成する作用として統一的に捉え、その作用がどの状態と
reachable future を生むかを計算できる理論へ再設計する。

AAT は、そのための幾何学的な土台を与える。software architecture を Atom と Law から構成される
geometry として定式化し、状態、lawful locus、局所性、貼り合わせ、obstruction、repair を
数学的対象にする。

SFT は、その幾何を software evolution の力学にする。artifact、practice、AI、review、CI、
operational feedback を進化へ作用する力として扱い、trajectory、reachable future、feedback、
governance を数学的対象にする。

AAT の幾何と SFT の力学を接続することで、software engineering を、architecture の状態と
software evolution の trajectory を定理と計算によって研究する学問へ進める。

### 1.2 プロジェクト全体が目指すもの

このプロジェクトは、次の連鎖を一つの研究計画として実現することを目指す。

```text
architectural fact を言葉にする
  -> AAT が architecture geometry を構成する
  -> Lean が数学的主張を形式化する
  -> ArchSig が有限な instance を計測する
  -> SFT が architecture geometry を evolution dynamics へ接続する
  -> trajectory、reachable future、feedback、governance を計算する
```

この全体計画を一文で表すなら、次のようになる。

> AAT の幾何によって software architecture を計算可能な対象にし、SFT の力学によって
> software evolution の trajectory と reachable future を計算可能にする。

### 1.3 AAT と SFT

大きな研究計画には、少なくとも二つの理論がある。

```text
AAT
  software architecture を geometry にする。
  state、Law、locality、gluing、obstruction、repair の計算基盤を与える。

SFT
  AAT geometry を software evolution の dynamics にする。
  force、trajectory、reachable future、feedback、governance を扱う。
```

AAT は architecture の構造と意味を幾何として構成する。SFT は、その幾何の上に、時間、変更、分岐、
合流、feedback、governance を含む進化の力学を構成する。幾何学的な状態空間から進化の trajectory へ、
software engineering の理論が積み上がる。

SAGA は、この長期的な研究計画の中で、architecture geometry の局所情報から大域状態を計算するための
cohomological invariant を確立した成果として発表する。ArchSig による有限 measurement は、
その幾何学的計算を実行可能な certificate として示す。

### 1.4 現場から生まれる独立研究

AAT は、software engineering の現場で繰り返し現れる問いから生まれた。

> 各局所領域で成立する repair は、どの条件で一つの大域 repair へ貼り合うか。

component ごとの実装、test、review、repair がそれぞれ成立している状況でも、architecture 全体の
整合性は局所情報の組合せに依存する。AAT は、この現象を局所データの貼り合わせ問題として捉え、
software architecture に固有の cohomology を構成する。

この発想は、現場の問題を新しい lint rule や checklist へ直ちに変換する代わりに、問題を生み出す
構造そのものを数学的対象として捉えるところから始まる。局所repairの大域化可能性という問いを立てると、
site、sheaf、cohomology が architecture の数学として自然に現れる。

独立研究は、問題設定、数学、形式化、実装を一人の研究系列として縦に接続できる。

```text
現場の architecture 問題
  -> Atom と Law による理論
  -> algebraic geometry
  -> SAGA theorem
  -> Lean formalization
  -> ArchSig measurement
```

FOSE 論文は、この vertical integration を研究の個性として示す。現場のエンジニアとして発見した問い、
独立研究者として育てた理論、形式化による数学的検査、実装による有限計測が、一つの provenance の中で
つながっていることを伝える。

論文が伝える面白さは、次の四点に置く。

- software architecture に cohomology が現れる意外性
- local repair の大域化という現場的な問いから数学が生まれること
- Atom と Law から architecture geometry を構成する理論的な野心
- Lean と ArchSig によって数学が証明と実行の両方へ進むこと

読者が構成を追うことで、最初の意外性が数学的な必然へ変わる。その体験を、FOSE 論文の語りの中心に置く。

## 2. 私たちがやりたいこと

### 2.1 architecture を数学的対象にする

私たちが実現したい最も根本的な変化は、software architecture を、図、説明文、経験則、
review 上の好みだけに依存する対象から、定義、構成、定理、反例、obstruction、certificate によって
研究できる数学的対象へ移すことである。

architecture を数学的対象にすることで、次のような問いを明確に立てられるようにしたい。

- どの architectural fact が対象に存在するか。
- どの Law が選択され、それらの連立方程式がどの lawful locus を切り出すか。
- 局所的に成立する構成は、大域的な architecture object へ貼り合うか。
- 選択された operation による repair は存在するか。
- repair が阻まれる場合、どの obstruction がそれを妨げているか。
- 局所repair が存在する場合、それらは一つの大域repairへ貼り合うか。
- architecture はどの方向へ変形でき、どの Law がその変形を妨げるか。
- 責務、補償、不変条件、業務意味などの semantic fact は、複数の component を越えて整合するか。

AAT は、これらを同じ数学的基盤上の問題として統合して扱う。

### 2.2 Atom と Law から始める

AAT の primitive は Atom である。Atom は、同じ意味を保つ最小単位となる、型付きの
primitive architectural fact である。

Atom は、component や dependency のような構造的事実と、contract、effect、state、authority、
responsibility、compensation、invariant、業務意味のような semantic fact を、選択された
vocabulary の中で同じ基礎単位として扱う。

architectural fact の供給は Atom が担う。Law は Atom family の上に方程式、関係、零点条件を与える。
複数の Law は連立方程式として働き、その共通零点が lawful locus を形成する。

```text
Atom family
  -> architecture object
  -> Law equations
  -> obstruction ideal
  -> lawful locus
```

この出発点により、AAT は programming language、framework、ADL、repository structure を越えて、
選択された architectural fact を共通の数学的基盤で扱える。

### 2.3 architecture を本物の代数幾何として扱う

AAT は、architecture の構成そのものに代数幾何を用いる。

- Law は方程式系として扱われる。
- Law failure の witness は ideal を生成する。
- lawful architecture は lawful locus として現れる。
- architecture context は site を構成する。
- 局所データは sheaf と section によって扱われる。
- 局所データの貼り合わせ不能性は cohomology class として現れる。
- repair と変形は deformation、singularity、derived structure へ接続される。

代数幾何を選ぶ理由は、複数の Law が同時に作用する構造、局所と大域、破れとrepair、
変形とobstructionを、一つの言語で扱えるからである。

### 2.4 選択された reading の中で確かに語る

AAT の主張は、選択された Atom vocabulary、Law universe、coverage topology、係数、witness、
measurement profile に相対的である。

この相対性は、何を対象にし、どの仮定の下で、何が従うかを明示する数学的な規律である。
選択された reading の中では定理を強く述べ、各主張を選択された対象に正確に対応させる。

計測済みの零と計測対象外を異なる状態として保持する。これは AAT の数学、Lean の形式化、
ArchSig の診断、将来の SFT による進化判断まで一貫して守るべき原則である。

### 2.5 AAT が目指す到達点

AAT は、architecture を記述する vocabulary を数学的構成へ高める。Atom と Law から
architecture geometry を構成し、software architecture 固有の存在問題、貼り合わせ問題、
repair問題、obstruction問題、変形問題を定理として解くことを目指す。

長期的には、次の対象が同じ理論系列に置かれる。

```text
lawful locus
  -> local section and gluing
  -> obstruction cohomology
  -> repairability and response geometry
  -> deformation and singularity
  -> higher / derived / stacky structure
  -> finite measurement
  -> SFT の evolution geometry への入力
```

この意味で、AAT は software architecture を数学的に研究するための長期的な理論計画である。
個々の theorem と analyzer は、その理論計画から生まれる成果として位置づけられる。

## 3. AAT の中での SAGA の位置

### 3.1 SAGA 以前に成立していたもの

SAGA 以前の AAT には、Atom、Law、architecture object、ideal、lawful locus、site、sheaf、
semantic repair complex、有限 obstruction class といった構成が存在していた。

これらによって、architecture を algebraic geometry の対象として構成し、局所repairの差を
有限な obstruction として読む基礎は得られていた。一方で、architecture semantics 側で構成された
semantic repair の `H¹` と、AAT site 上の Čech `H¹` を同じ由来を持つ対象として接続することが、
研究上の主要課題として残っていた。

### 3.2 SAGA が達成したこと

SAGA は、AAT における semantic repair の局所―大域問題を、AAT 自身の Atom、Law、site、cover、
law-generated coefficient から構成される cohomology へ接続する。

中心となる連鎖は次である。

```text
Atom
  -> Law equation
  -> law witness ideal
  -> law-generated coefficient
  -> AAT site and selected cover
  -> local semantic repair
  -> overlap residual
  -> semantic repair additive H¹
  -> cover-relative Čech H¹
```

この連鎖により、局所repairの族が一つの大域repairへ貼り合うかという問いを、residual class の
零性として扱える。さらに semantic repair 側の `H¹` と AAT site 側の cover-relative Čech `H¹` の
比較により、architecture semantics と AAT geometry の二つの読みが接続される。

SAGA により、AAT は architecture の記述理論から、architecture 固有の局所―大域 obstruction を
定理として扱う理論へ進んだ。

### 3.3 SAGA をマイルストーンとしてどう定義するか

本論文では、SAGA を次のように位置づける。

> SAGA は、Atom と Law から構成される AAT が、局所的な semantic repair の大域化を
> cohomological obstruction として扱える段階へ到達したことを示す、AAT の局所―大域マイルストーンである。

この位置づけには三つの意味がある。

1. **architecture-specific な cohomology**

   AAT の Atom、Law、site、cover、coefficient、repair semantics が同じ provenance の中で
   接続され、architecture 固有の cohomology を構成する。

2. **局所から大域へ進む理論能力**

   各局所領域で成立するrepairの大域化可能性を判定し、個別の Law failure から局所―大域
   obstruction へ解析対象を広げる。

3. **後続研究の基礎**

   G-08 の operation response、repairability、minimal circuit、response-kernel descent、さらに
   deformation、higher obstruction、derived / stacky structure へ進むための descent engine を与える。

### 3.4 Lean の役割

Lean は、SAGA の数学的主張、仮定、依存、比較写像、零判定を形式的に検査する。

FOSE 論文では、論文中の各主要 claim を具体的な Lean declaration と対応させる。仮定の下で成立する
比較定理、AAT 入力から構成される instance、非零性を示す finite example を区別し、それぞれの
証明状態を明記する。

Lean 形式化は、SAGA の定理の強さ、使用された仮定、comparison data の由来、非自明性を
検査可能にする研究上の構成要素である。

### 3.5 ArchSig の役割

ArchSig は、AAT の有限 measurement contract を実行する計測層である。

SAGA に対しては、選択された有限 Atom、Law、cover、repair data から、residual、`B¹` membership、
class の零・非零、comparison、assumption ledger、source-grounded support を計算し、再検証可能な
certificate として出力する役割を担う。

```text
AAT / Lean
  generic construction and theorem
        ↓ finite measurement contract
ArchSig
  selected instance measurement and certificate
```

したがって FOSE 論文における ArchSig は、SAGA が実行可能な有限数学であり、実コードから選択された
architecture evidence へ接続できることを示す measurement implementation として位置づける。

### 3.6 SAGA の後に続く研究

SAGA は、AAT の長期的な研究計画を構成する局所―大域段階である。

```text
Atom and Law
  -> architecture geometry
  -> lawful locus, site, sheaf
  -> SAGA: semantic repair descent and H¹ obstruction
  -> G-08: operation response, local repairability, circuit, kernel descent
  -> deformation, singularity, higher / derived / stacky obstruction
  -> SFT: architecture projection を用いた software evolution theory
```

SAGA の発表では、SAGA によって確立された局所―大域段階を中心に据え、その達成が後続研究に
何を可能にしたかを展望として説明する。
