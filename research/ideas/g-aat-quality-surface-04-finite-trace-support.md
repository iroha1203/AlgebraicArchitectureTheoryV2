# Finite Trace Support

## 主張

`support : List Atom` を ordered finite input geometry として受け取り、その各 atom を読む canonical source-trace probe family と finite-support trace shadow を作る。support trace vector と、four-bit layer と support trace vector から生成される observation は、この finite-support trace shadow を通じて factor する。

## 候補種別

`target-refinement`

## 依拠

- `SemanticRepairTraceProbeShadow.lean`: finite trace-probe shadow と probe-generated observation factorization
- `SemanticRepairTraceAwareShadow.lean`: singleton trace-aware shadow
- `SemanticRepairFiniteShadowSeparation.lean`: Cycle 13 source-trace witness

## 非自明性

Cycle 15 は任意の supplied finite probe family を扱った。本候補は、その probe family を finite atom support から生成される canonical probes に特殊化し、Cycle 15 の generic trace-probe shadow と finite-support shadow の compatibility を明示する。これは full completeness theorem ではなく、probe boundary を監査しやすくする refinement である。

## 数学的興味

finite trace-probe shadow の実用的な boundary では、任意関数 probe ではなく、どの atom support を読むかが重要になる。本候補は、support list の順序と重複をそのまま持つ有限入力幾何として trace vector を定義し、support-generated observation に限定した factorization を証明する。

## GOAL への前進

G-04 の finite trace adequacy 周辺で、arbitrary supplied probe family と finite atom support の間を Lean theorem として接続する。ただし finite probe completeness certificate、admissible observation theorem、full representation adequacy は未完のまま残す。

ADL、静的解析、metric dashboard、AI review は source reference / trace atom の有限集合を保持することが多い。本候補は、その有限 atom support を AAT 側の trace-probe shadow に写す canonical specialization である。
