# G-aat-quality-surface-03 — relation-atom geometry による AAT 型 DAG / mixed-atom analysis

- `id`: `G-aat-quality-surface-03`
- `status`: `draft`
- `research mode`: `score-phase`
- `predecessor`: `G-aat-quality-surface-02` の target theorem proof 後の NEXT STEP として読む。02 が semantic repair-gluing descent を証明するまでは active にしない。
- `research aim`: relation atom を、通常の dependency edge / DAG / component relation の記録ではなく、AAT の finite algebraic-geometric object として扱う。relation atom family から path law、incidence locus、relation cover、relation obstruction、repair / refactor direction、projection nonfaithfulness を読み、さらに semantic atom、law atom、trace atom、component atom と組み合わせて、ADL / 静的解析 / AI review が返す依存グラフを越えた品質サーフェイスを作る。狙いは「DAG を解析できる」ではなく、DAG 的 relation fragment を AAT の lawful locus / obstruction / gluing / deformation / mixed-atom certificate の対象にすることである。
- `core tension`: relation atom だけを見ると同じ DAG、同じ topological order、同じ reachability に見える architecture が、semantic obligation、law profile、source trace、repair frontier と組み合わさると、どの条件で別の幾何点、別の obstruction class、別の repair basin として分離されるのか。逆に、semantic / law / trace の差が relation geometry 上で可視化されない場合、どの projection が何を忘れているのか。AAT らしさは、relation atom を既存 DAG 解析の言い換えにせず、relation law の解空間、path ideal、incidence / degeneracy locus、mixed-atom gluing failure として読めるかにある。
- `rival`: primary rival は dependency graph / call graph / import graph を扱う静的解析器と architecture conformance checker である。これらは cycle detection、topological order、reachability、layer violation、unused dependency、module boundary violation を高速に出せる。secondary rival は ADL / architecture description language であり、component、connector、view、constraint、declared architecture との一致を扱える。strong rival は無限コンテキスト AI code-review agent であり、依存グラフ、責務、変更履歴、自然言語意図を読んで refactor plan を提案できる。この GOAL は、それらが得意な relation-level signal を尊重したうえで、relation atom と semantic / law / trace atom の混合が作る obstruction、nonfaithfulness、repair geometry、local-to-global failure を AAT の theorem / finite witness / computable certificate として固定することで差分を作る。
- `novelty hypothesis`: 既存の DAG 解析は relation 構造を上手く扱うが、relation law の解空間として architecture を読み、edge / path / incidence の同値や変形が semantic obligation、law profile、traceability、repair potential をどう変えるかまでは一つの数学対象にしにくい。AAT では relation atom を primitive fact として持つため、DAG は単なる可視化ではなく、mixed atom certificate space の一つの projection である。この projection が faithful な場合、既存 DAG 解析を AAT 内に取り込める。faithful でない場合、AAT はどの semantic / law / trace obstruction が落ちたかを theorem として示せる。
- `research lift`: 品質分析を「依存グラフに違反があるか」から「relation atom family と他 atom family が作る解空間のどの locus にいるか」へ移す。DAG 上の local fix、edge deletion、layer repair、topological reordering が、semantic responsibility、law satisfaction、source trace、repair frontier と整合するかを、mixed-atom certificate と obstruction として扱えるようにする。
- `claim boundary`: 有限 Atom vocabulary、relation atom vocabulary、finite directed acyclic relation family、supplied edge / path / incidence reading、law profile、coverage topology、semantic atom vocabulary、trace field、repair operation family、certificate selector、verdict discipline 上で語る。AAT 側は ArchMap / observation layer が供給した relation atoms と source references に相対化する。source extraction completeness、call graph completeness、runtime causality、dynamic trace completeness、AI の理解限界、コード全体の品質判定、一般グラフ理論全体の包摂は主張しない。
- `capability categories`: relation-atom-geometry、dag-lawful-locus、path-law、incidence-obstruction、mixed-atom-gluing、semantic-relation-coupling、law-relation-coupling、traceability、projection-nonfaithfulness、repair-basin、computability、invariance、rival-integration。
- `threshold policy`: active 化する場合、active SCORE threshold は tracking Issue の threshold state で管理する。03 は 02 の target theorem proof 後に始める次フェーズ候補であり、GOAL カードには固定 threshold を置かない。threshold は、relation atom の基礎定義だけでなく、少なくとも一つの finite DAG witness、mixed-atom nonfaithfulness result、repair / obstruction bridge、rival comparison が揃う水準に置く。
- `portfolio constraint`: phase completion までに、少なくとも relation-atom-geometry、mixed-atom-gluing、projection-nonfaithfulness、repair-basin の 4 カテゴリで正の SCORE を持つこと。少なくとも 1 件は relation-only DAG fragment を AAT 内で faithful に読む結果、少なくとも 1 件は relation projection が semantic / law / trace obstruction を忘れる nonfaithfulness result、少なくとも 1 件は Lean proved または finite witness として固定すること。最終 SCORE の 50% 以上を cycle detection、topological order、reachability、layer violation の言い換えだけで占めてはならない。
- `phase boundary criteria`: `research/reports/G-aat-quality-surface-03.md` が coherent な report section / paper seed として読めること。relation atom を finite DAG / path law / incidence object として読む基礎語彙、relation-only faithful fragment、mixed-atom nonfaithfulness theorem、relation repair が semantic / law / trace atom と衝突する finite witness、ADL / static analyzer / AI agent に対する phase delta、次に狙う theorem frontier が揃っていること。
- `reward rubric`: base score は relation atom を AAT の幾何対象としてどれだけ使えるようにしたか、また relation atom と他 atom の混合がどれだけ新しい分析面を作ったかで決める。`0`: 既存 DAG 解析、cycle detection、topological sort、reachability、layer violation の言い換え。`10-20`: relation atom、edge reading、path notation、DAG profile の補助定義。`30-40`: finite relation atlas、path law、incidence predicate、relation certificate、relation repair operation の一部を与える結果。`50-70`: relation atom と semantic / law / trace atom の二者以上を接続し、projection faithfulness / nonfaithfulness、repair-hit necessity、trace-preserving relation repair を示す bridge。`80-100`: relation-only DAG 解析を AAT 内に統合しつつ、それだけでは取れない mixed-atom obstruction、degeneracy locus、repair basin、local-to-global failure、rival separation を theorem / finite witness として固定する結果。`genius` 候補は、relation atom geometry が semantic repair-gluing、lawfulness、traceability、AI / ADL rival separation を一つの定理的構造へ圧縮する場合だけ検討する。
- `dullness filter`: 既存 dependency graph / call graph / import graph の可視化、cycle detection、topological order、reachability、layer violation、unused dependency、DAG metric を AAT 語彙で言い換えただけの候補を弾く。relation atom を単なる edge label として使い、lawful locus、path law、incidence obstruction、mixed-atom certificate、semantic / law / trace projection のいずれにも接続しない候補を弾く。ADL の connector / view / constraint や AI review の依存指摘を AAT 語彙に翻訳しただけの候補を弾く。
- `frontier`: relation atom family、finite DAG atlas、path law、path ideal、incidence algebra、quiver-style representation、poset sheaf、relation cover、relation lawful locus、relation obstruction class、edge deletion / edge contraction / reordering repair、relation-semantic obstruction、relation-law obstruction、relation-trace nonfaithfulness、mixed-atom certificate space、DAG projection faithfulness theorem、relation repair basin、source-ref relation trace witness、ADL / static analyzer / AI rival separation。
- `spine`: まず relation atom vocabulary と finite DAG profile を定め、edge、path、incidence、reachability、order relation を AAT の relation certificate として置く。次に relation law の lawful locus と projection を定め、既存 DAG 解析が faithful に読める fragment を切り出す。続いて semantic atom、law atom、trace atomを加え、同じ relation DAG でも mixed certificate が分離される finite nonfaithfulness witness を作る。最後に relation repair operation が semantic / law / trace obstruction を hit する条件を定理または finite witness として固定し、relation atom と他 atom を組み合わせた品質サーフェイスとして report にまとめる。
