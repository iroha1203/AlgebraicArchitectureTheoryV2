# Schema Cheatsheet

This file summarizes the final ArchMap schema and the four authoring sibling
schemas used by the redesigned workflow.

## ArchMap v0.5.0

ArchMap itself is unchanged by the authoring workflow.

Top-level fields:

- `schema`: `archmap/v0.5.0`
- `id`
- `extractionDoctrineRef` (optional; defaults to `doctrine:aat-canonical@1`)
- `sources`
- `atoms`
- `contexts`
- `covers`

Atom shape:

```json
{
  "id": "atom:capability:orders.Handler:handlesCommand:PlaceOrder",
  "kind": "capability",
  "subject": "orders.Handler",
  "axis": "static",
  "predicate": "handlesCommand",
  "object": "PlaceOrder",
  "refs": ["src:src/orders/handler.rs"]
}
```

Context shape:

```json
{
  "id": "ctx:orders-command-path",
  "label": "orders command path",
  "atoms": ["atom:capability:orders.Handler:handlesCommand:PlaceOrder"],
  "refs": ["src:src/orders/handler.rs"],
  "restrictsTo": ["ctx:orders-shared-boundary"]
}
```

Cover shape:

```json
{
  "id": "cover:orders-cech",
  "label": "orders Cech measurement surface",
  "contexts": ["ctx:orders-command-path", "ctx:orders-shared-boundary"],
  "refs": ["src:docs/architecture/orders.md"]
}
```

Validate with:

```bash
archsig archmap --input archmap.json --out archmap-validation.json
```

## archmap-candidate-packet/v0.5.0

Emitted by each reading pass and chunk.

`privateUnavailableNotes` contains sanitized summaries only. Do not place local
absolute paths, personal names, nonpublic labels, secrets, or
machine-specific identifiers in this field.

```json
{
  "schema": "archmap-candidate-packet/v0.5.0",
  "id": "candidates:pass-a:chunk-01",
  "scopeManifestRef": "scope:orders-service",
  "passId": "pass-a",
  "chunk": { "worklistOrderFrom": 1, "worklistOrderTo": 20 },
  "reviewedSources": ["src:src/orders/handler.rs"],
  "candidateSources": {
    "src:src/orders/handler.rs": {
      "kind": "file",
      "path": "src/orders/handler.rs"
    }
  },
  "candidateAtoms": [],
  "candidateContexts": [],
  "candidateCovers": [],
  "surveyRows": [
    {
      "sourceId": "src:src/orders/handler.rs",
      "status": "read",
      "surveyedKinds": ["component", "capability", "semantic"],
      "candidateAtomIds": [],
      "notes": []
    },
    {
      "sourceId": "src:private/runtime.log",
      "status": "skipped",
      "reason": "private",
      "surveyedKinds": [],
      "candidateAtomIds": [],
      "notes": []
    }
  ],
  "privateUnavailableNotes": [],
  "selfReview": {
    "notScriptGenerated": true,
    "notCoarseWhenEvidenceWasRicher": true,
    "semanticAtomsHaveUseEvidence": true,
    "noDiagnosticShortcutAtoms": true,
    "worklistChunkFullyRead": true,
    "aliasPreservingSemantics": true
  }
}
```

`partial` and `skipped` rows must include one procedural `reason`: `private`,
`binary`, `unreadable`, or `tooling-error`. `selfReview` fields are validation
gates; a false field means the packet is not ready for consistency comparison.

## archmap-extraction-consistency/v0.5.0

Mechanical diff output plus integrator adjudication. `adjudications` is empty in
the mechanical output and is filled only after source rereading.

```json
{
  "schema": "archmap-extraction-consistency/v0.5.0",
  "id": "consistency:orders-service",
  "scopeManifestRef": "scope:orders-service",
  "passARefs": ["candidates:pass-a:chunk-01"],
  "passBRefs": ["candidates:pass-b:chunk-01"],
  "atomMatchKeySpec": "kind | NFC(trim(subject)) | axis | predicate? | object?",
  "matched": {
    "count": 12,
    "rows": [
      {
        "key": "semantic|orders.Handler|semantic|uses|cancel path",
        "passAAtomIds": ["atom:pass-a:cancel-path"],
        "passBAtomIds": ["atom:pass-b:cancel-path"],
        "refs": ["src:src/orders/handler.rs", "src:src/orders/handler.rs:42"]
      }
    ]
  },
  "onlyInPassA": [],
  "onlyInPassB": [],
  "matchRate": 1.0,
  "contextDiff": { "matched": 2, "onlyInPassA": [], "onlyInPassB": [] },
  "adjudications": []
}
```

## archmap-coverage-ledger/v0.5.0

Selected-scope survey record.

```json
{
  "schema": "archmap-coverage-ledger/v0.5.0",
  "id": "coverage:orders-service",
  "scopeManifestRef": "scope:orders-service",
  "archmapRef": "archmap:orders-service",
  "passRefs": ["pass-a", "pass-b"],
  "rows": [
    {
      "sourceId": "src:src/orders/handler.rs",
      "surveyStatus": "surveyed",
      "passes": ["pass-a", "pass-b"],
      "surveyedKinds": ["component", "capability", "semantic"],
      "adoptedAtomIds": []
    }
  ],
  "claimBoundary": "Rows record the authoring survey of the selected scope at the recorded revision. They do not assert extraction completeness."
}
```
