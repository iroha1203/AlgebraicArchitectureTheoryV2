# Archived Tool Docs Before ArchMap Primary

This directory is a historical archive of the pre-ArchMap-primary `docs/tool/` documents.

These files are not current source of truth. They are retained only to preserve the design history that existed before Issue #1203 and Issue #1204 moved ArchSig toward an ArchMap-primary workflow and moved SFT / operational forecast surfaces to FieldSig.

Current references:

- `docs/tool/README.md` is the minimal current entry point.
- `tools/archsig/README.md` and `tools/archsig/docs/` describe the current ArchSig surface.
- `tools/fieldsig/README.md` and `tools/fieldsig/docs/` describe current FieldSig surfaces.

Historical reading boundary:

- Statements in archived files about `docs/tool/` being the source of truth are stale.
- Statements in archived files presenting Lean / Python import graph scan or Sig0 as ArchSig Core are stale.
- Archived PRDs, reports, roadmap notes, and protocol drafts may mention surfaces that have moved to FieldSig or been replaced by the ArchMap-primary boundary.
- The archive does not update claim boundaries, schema compatibility policy, or command examples for current behavior.

Use this archive only when investigating why older docs or Issues used scan-first, Sig0-first, mixed ArchSig / FieldSig, or operational-feedback wording.
