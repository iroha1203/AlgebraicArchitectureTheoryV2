Install and prerequisites

ArchSig currently runs from the repository tooling crate. You need Rust and Cargo available, then run commands from the repository root.

The default scan supports Lean repositories. Python mode supports static import extraction with the standard AST parser.

Fixture scan

Begin with the minimal fixture so the artifact shape is visible before live repository noise appears. The fixture includes policy input and runtime edge evidence.

Validate Sig0

Validation checks an existing Sig0 file. It does not rescan source files and it does not strengthen an unmeasured axis into a measured claim.

Repository scan

After the fixture, scan your repository with the smallest useful configuration. Add policy and runtime inputs only when you know which evidence they should represent.

First reading

  • componentsThe selected component universe. Ask whether names match the architecture boundary you expected.
  • edgesThe observed dependency relation. Ask whether surprising edges are real dependencies or adapter limitations.
  • signatureThe architecture axes emitted for this universe.
  • metricStatusThe field that tells you whether a value was measured, unavailable, unsupported, private, or out of scope.

A zero value is useful only when its status says the axis was measured.

Next moves

If you want PR review, continue to CI and PR Review. If you want to understand reports first, read Reading Output. If you need to prepare policy, runtime edges, PR metadata, or proposal inputs, read Inputs and Configuration.
