First adoption

First adoption establishes a baseline without pretending that unsupported axes are safe. The useful result is a first report with unmeasured axes and unsupported constructs made explicit.

A good first adoption run is not a pass/fail ceremony. It is a vocabulary alignment exercise: teams learn which axes ArchSig can already observe, which local policies need explicit input, and which framework behaviors still sit outside the adapter.

PR review

PR review compares before and after artifacts, then turns the difference into AIR, theorem precondition checks, Feature Extension Report, policy decision, and PR comment summary.

The reviewer should be able to trace every warning back to a measured witness, a policy threshold, a missing precondition, or an unmeasured axis. A workflow that cannot explain this trace has lost the main value of Architecture Signature.

Generated change

AI-generated change provenance records provider, model, prompt ref, generated patch, and human review boundary. The provenance record supports review and reproducibility; it does not decide whether the change is good or bad.

Generated-change workflow is especially sensitive to boundary erasure. The prompt may describe a desired operation, the patch may implement only part of it, and the review may accept a narrower claim. ArchSig keeps those layers separate.

  • Prompt and generated patch Evidence refs for the generated action and its proposed scope.
  • Human review boundary Whether the generated change was reviewed, and what review evidence is available.

Scheduled scan

Scheduled scans update drift ledgers and operational feedback artifacts. Daily drift is a trend artifact, not proof that a report warning caused an operational outcome.

The scheduled workflow is useful because it makes gradual architecture movement visible before a single PR looks responsible. It supports calibration and ownership review, not causal attribution by itself.

Dynamics and forecast

Architecture Dynamics and SFT forecasting workflows hold finite support, bounded horizon, proposal refs, unknown remainder, and review recommendations apart from probability or causal forecast claims.
