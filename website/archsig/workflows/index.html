First adoption

  1. Scan one repositoryProduce Sig0 and validation without custom policy.
  2. Add policy selectivelyEncode one real boundary or law reference rather than a complete architecture constitution.
  3. Compare one PRRun before/head snapshots and signature diff on a change the team already understands.
  4. Review the report manuallyCheck whether the signal helped reviewers inspect the right files and assumptions.

PR review

The PR workflow is the daily path: scan base, scan head, validate both, compare signatures, generate AIR, check theorem preconditions, produce a feature report, evaluate policy, and post a bounded review summary.

Do not make the first rollout a hard gate. Start with visible comments and a small set of explicit organization thresholds.

Generated change

For AI-generated code, preserve the proposal and generation context as an artifact. The review question is not only whether tests pass, but whether the generated change moved a boundary or responsibility without making that movement explicit.

  • InputAI proposal, prompt summary, linked issue, touched files, and generated-change provenance.
  • ObservationStructural diff, semantic AIR, changed responsibilities, and missing confirmation.
  • ReviewHuman confirmation for semantic claims and explicit treatment of unmeasured runtime behavior.

Scheduled scan

Scheduled scans help teams notice architecture drift outside individual PRs. They should produce snapshots, trajectory reports, drift ledgers, and daily outcome records. The value is trend and calibration, not a sudden universal quality score.

Proposal forecast

SFT-facing workflows start from descriptors and move through operation support estimates, forecast cone skeletons, consequence envelopes, calibration hooks, AI proposal governance, and SFT forecast reports. These artifacts describe bounded reachable futures and review pressure; they do not make point predictions.

Feedback loop

The loop closes when review outcomes, incidents, repairs, ownership changes, and calibration decisions feed back into future thresholds and interpretation. This is how ArchSig turns one-off PR comments into an architecture evidence layer.
