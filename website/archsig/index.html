What you can do with ArchSig

ArchSig is built for architecture review where the important evidence is not only in imports. It separates observation, policy, analysis, and SFT handoff so each artifact can be read without turning tooling output into theorem truth.

  • Record source-grounded architecture observationsArchMap stores atom, molecule, semantic observation, concern, gap, provenance, and non-conclusion records read from code, docs, tests, config, traces, or PR context.
  • Select the law universe explicitlyLawPolicy chooses witness rules, molecule patterns, obstruction definitions, signature axes, coverage requirements, exactness assumptions, optional ACTS spectrum measurement profiles, and optional homotopy path / filler / loop profiles for one review context.
  • Compute law-relative analysisArchSig emits an archsig-analysis-packet-v0 with molecule readings, obstruction circuits, signature axes, flatness reading, ArchitectureSpectrumReport hotspots, ArchitectureHomotopyReport loop / hole / holonomy readings, repair candidates, coverage gaps, and LLM interpretation notes.
  • Hand off bounded state to FieldSigFieldSig reads the analysis packet as local AAT algebra state, not as forecast truth or global safety.

5-minute path

The first successful session should run the LLM-native workflow on a fixture, then inspect the analysis packet before reading downstream report projections.

  1. Validate ArchMap observationCheck source refs, atom observations, molecule observations, semantic observations, concern hints, gaps, and non-conclusions.
  2. Validate LawPolicyCheck the selected law universe, witness rules, obstruction definitions, signature axes, coverage requirements, and exactness assumptions.
  3. Build the analysis packetRead law-relative obstruction circuits, signature axis values, flatness reading, repair candidates, and coverage gaps.
  4. Read projections secondAIR, theorem checks, feature reports, and observable bundles are downstream review projections from the analysis packet.
  5. Send only bounded state to FieldSigUse FieldSig for operation support and forecast surfaces; do not read raw ArchMap as SFT truth.

Choose your path

  • New to ArchSigRun the LLM-native workflow and learn the shape of ArchMap, LawPolicy, and the analysis packet.
  • Reviewing a PRWire analysis packet generation and bounded report projections into CI.
  • Using ArchMapRead source-grounded Atom observations without promoting them to lawfulness or obstruction truth.
  • Reading JSON outputRead analysis packets and report projections as review maps, not raw data dumps.
  • Setting up team workflowsAdopt ArchSig for LLM observation, selected policy analysis, generated changes, and FieldSig handoff.
  • Using AIRRead AIR as a bounded projection from analysis state into existing review surfaces.
  • Looking for commandsUse the reference index when you already know the artifact you need.

Product position

ArchSig is an AI-native Architecture Evidence Layer for PR review. It turns source-grounded observations and selected laws into structured analysis packets, then projects them into review artifacts without claiming global architecture truth.

ArchSig helps reviewers see what was observed, which laws were selected, what obstruction or signature readings appear, and where to inspect next.
