Primary workflow

analyze is the primary ArchSig workflow entry. It consumes archmap/v1 and law-policy/v1, then writes validation reports, normalized atoms, typed evaluator results, architecture-distance.json, summary, viewer data, and optional raw v1 artifacts.

archsig-analysis-summary.json is emitted by analyze as the compact first reading surface. It includes distanceInsights, distanceDiagnosis, richPacketRefs, richDominantFindings, richReadingGuide, and an actionQueue over generated, spectrum, homotopy, and structural refs.

Workflow outputs

  • archmap-validation.jsonValidation report for the supplied archmap/v1.
  • law-policy-validation.jsonValidation report for the selected law-policy/v1.
  • normalized-archmap.jsonArchMap atoms normalized into the ArchSig typed model.
  • typed-evaluator-results.jsonEvaluator statuses, support refs, basis refs, and bounded conclusions.
  • architecture-distance.jsonThe primary archsig-architecture-distance/v1 artifact for Part IV family summaries, distanceInsights, distanceDiagnosis, homotopy rows, curvature rows, and representation metric rows.
  • archsig-analysis-summary.jsonThe compact first reading surface with the same distance insights and diagnosis.
  • archsig-atom-viewer-data.jsonBounded viewer projection with the same rich packet refs and report-pane reading guide as the summary.
  • archsig-run-manifest.jsonRun artifact manifest and generated / omitted artifact state.
  • archsig-analysis-packet.jsonThe raw archsig-analysis-packet/v1 detail store, emitted with --emit-raw-artifacts.
  • archsig-analysis-validation.jsonValidation report for the generated v1 analysis packet.
  • llm-interpretation-packet.jsonThe typed evaluator packet serialized for LLM reading when raw artifacts are emitted.

Command surfaces

CommandUse it for
archmapValidate an archmap/v1.
law-policyValidate a selected law-policy/v1 evaluator selector.
analyzeRun the primary ArchMap plus LawPolicy analysis workflow.
pr-reviewRead a base ArchMap, PR-local ArchMap delta, and LawPolicy for small PR review.
schema-catalogEmit the current ArchSig schema catalog.

--strict-distance rejects missing profile selectors, blocked typed support, hidden missing families, and partial canonical distance-family bundles instead of treating them as measured zero.

Removed runtime surfaces

  • llm-native-workflow / north-star-workflowRemoved as runtime aliases. Use analyze.
  • interpretation-profileRemoved as a runtime alias. Use law-policy with law-policy/v1.
  • archsig-analysis / aat-analysisRemoved as v0 packet-builder runtime surfaces. Use analyze --emit-raw-artifacts.
  • analysis-summary / summaryRemoved as standalone runtime surfaces. Read the summary emitted by analyze.
  • archmap-generate / codebase-inspectionRemoved from the v1 runtime contract.

Reference boundaries

  • Current input contractArchSig analysis consumes archmap/v1 plus law-policy/v1. v0 dual reader and legacy aliases are not v1 completion requirements.
  • PR input contractpr-review reads base ArchMap, PR-local ArchMap delta, and LawPolicy. Raw diff is not an ArchSig PR-review input.
  • Validation boundaryValidation checks artifact shape and guardrails. It is not a Lean proof, runtime correctness proof, or repair safety certificate.
