Clean extension

A clean coupon extension adds feature behavior behind an existing payment port. Structural evidence shows new feature components and no direct reach into payment internals. Semantic evidence says discount calculation is expressed through an explicit contract. The feature report can mark the extension as reviewable with no hidden dependency witness.

Hidden interaction

A hidden interaction appears when the feature reaches into an internal adapter cache or changes the order of payment normalization. The signal is not merely that a file imported another file. The signal is that a feature extension now depends on payment semantics without an explicit contract.

Semantic witness

A semantic witness records meaning that static edges alone do not capture: discount before tax, normalized amount assumptions, state transition ownership, or responsibility movement between Pricing, Coupon, and Payment. Semantic witnesses belong in AIR with explicit evidence boundaries.

PR comment anatomy

  • SummaryOne sentence naming the architectural movement.
  • EvidenceFiles, edges, semantic observations, runtime refs, or policy rules.
  • Reviewer actionWhat to inspect or change before merge.
  • BoundaryWhat was not measured, not confirmed, or not promoted to theorem status.

Dynamics reading

Architecture dynamics artifacts describe movement over time: drift ledgers, trajectory reports, operation proposal logs, repair adoption, and incident correlations. They are useful because they turn repeated review signals into calibration pressure.
