Core commands

  • llm-native-workflowValidate ArchMap, validate LawPolicy, build the ArchSig analysis packet, and emit bounded review projections.
  • archmapValidate supplied archmap-observation-map-v0 JSON.
  • law-policyValidate or emit a selected law-policy-v0 artifact.
  • archsig-analysisBuild archsig-analysis-packet-v0 from ArchMap + LawPolicy.
  • adapter-scanEmit bounded Lean / Python Sig0 adapter evidence when import-graph evidence is needed.

Review commands

  • validate-air, theorem-check, feature-reportRead projected review artifacts without promoting them to Lean theorem discharge.
  • aat-observable-bundleEmit or validate an AAT-facing observable bundle projection.
  • archmap-workflow, air-from-archmapCompatibility projection commands for existing AIR/report consumers; not the current source of truth.
  • policy-decision, pr-comment, baseline-suppressionReview-surface commands for policy decisions, PR comments, and accepted-risk metadata.

Policy and schema

  • organization-policyValidate organization policy shape.
  • report-artifactsValidate report retention manifests.
  • law-policy-templates, custom-rule-plugins, measurement-unitsValidate extension registry artifacts.

FieldSig handoff

SFT forecast, IntentMap, workflow evidence, governance, calibration, operational feedback, and dynamics commands belong to tools/fieldsig. The current handoff is archsig-analysis-sft-input: it reads archsig-analysis-packet-v0 and projects bounded obstruction, signature, repair, and coverage-gap refs into operation-support-estimate-v0.

  • archsig-analysis-sft-inputProject an ArchSig analysis packet into SFT operation-support input.
  • artifact-descriptorNormalize PRD, spec, issue, or AI proposal artifacts.
  • forecast-cone-skeleton, consequence-envelopeCreate bounded forecast review surfaces from selected FieldSig inputs.
  • forecast-calibration-hook, ai-proposal-governance, sft-forecastConnect proposals to governance and calibration artifacts.

Validation commands

Validation commands answer different questions. Use the narrowest validator for the artifact you already have.

  • archmapChecks source refs, atom observations, concern hints, gaps, and non-conclusions.
  • law-policyChecks selected laws, witnesses, obstruction definitions, axes, coverage, and exactness assumptions.
  • archsig-analysisChecks packet refs, obstruction links, signature refs, repair guardrails, coverage gaps, and non-conclusions.
  • validateChecks Sig0 adapter evidence when adapter output is used.

Source reference

The complete repository command guide remains the implementation-facing source for command examples and edge-case notes: tools/archsig/docs/commands.md and tools/fieldsig/docs/commands.md. This public page keeps the product-manual index: what each command consumes, what it emits, and when an engineer should choose it.

Command reading rule

A command produces an artifact. The artifact becomes trustworthy only when its schema, evidence, status fields, and review follow-ups are read together.
