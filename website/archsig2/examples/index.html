Payment boundary erosion

  • Returned findingThe change makes checkout code depend on payment adapter internals instead of the public payment contract.
  • Review actionInspect the checkout call path, payment adapter cache access, and the contract that should own payment state.
  • Repair candidateMove cache access behind a PaymentPort-style boundary or add an explicit payment-state query contract.
  • Confirmation checkVerify tests or traces that show checkout behavior does not depend on provider-specific cache state.

Coupon feature extension

  • Returned findingThe coupon feature is locally correct, but it shares rounding and eligibility assumptions with payment and pricing code.
  • Review actionInspect the feature extension path, shared semantic assumptions, and source refs for pricing and payment behavior.
  • Repair candidateIntroduce an explicit coupon/pricing contract before letting coupon logic read payment internals directly.
  • Confirmation checkAdd or inspect tests that fix rounding order, discount eligibility, and payment finalization behavior.

Operation order sensitivity

  • Returned findingThe paths reach similar endpoints in local code, but selected continuation readings differ on state finalization and retry behavior.
  • Review actionCompare the operation sequence, endpoint state, retry path, and compensation behavior before treating the paths as equivalent.
  • Repair candidateMake ordering constraints explicit in the workflow contract or split operations that should not commute.
  • Confirmation checkCollect source, test, or runtime evidence that the intended operation order is enforced.

Coverage repair queue

  • Returned findingArchSig cannot read several zero or nonzero conclusions because route coverage, provider traces, or ownership evidence is missing.
  • Review actionCollect the missing source refs, tests, docs, traces, or policy decisions before using the reading for repair planning.
  • Repair candidateRepair the ArchMap first: add measured evidence or record explicit unavailable/private/out-of-scope gaps.
  • Confirmation checkRerun the Skills after evidence repair and compare the summary's coverage gaps and action queue.
