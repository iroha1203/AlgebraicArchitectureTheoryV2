Hidden dependency witness

  • Returned findingThe clean path is CouponService -> declaredPaymentPort -> PaymentApi. The risky path adds CouponService -> PaymentAdapter.internalCache, which is a hidden dependency witness.
  • Why ArchSig sees itThe signal is not just an import edge. It is a feature extension that no longer factors through the declared payment interface under the selected LawPolicy.
  • Review actionOpen the coupon service, declared payment port, payment adapter, and internal cache refs before approving the extension.
  • Repair candidateRoute the needed cache-backed behavior through the declared port or add a new explicit payment query contract.

Static-flat, semantic obstruction

  • Returned findingThe static skeleton can be split while the semantic reading remains nonzero: round(tax(discount(subtotal))) and round(discount(tax(subtotal))) do not necessarily preserve the same amount or receipt behavior.
  • Why ArchSig sees itArchSig can carry semantic observations alongside source refs, so the review is not limited to whether the dependency graph looks clean.
  • Review actionInspect pricing, coupon, tax, rounding, receipt amount, and payment finalization tests together.
  • Confirmation checkRecord exact rounding and tax-order evidence before treating the repaired static shape as architecture-flat.

Repair transfer risk

  • Returned findingThe repair decreases one selected axis, but an operation delta shows transferred pressure on another axis such as runtime ordering, retry behavior, or state reconciliation.
  • Why ArchSig sees itThe packet includes repair candidates, operation deltas, decreased axes, transferred obstructions, side-effect axes, and operation readiness readings.
  • Review actionDo not merge the refactor as a pure cleanup until the target axis and transferred axis are both inspected.
  • Confirmation checkAsk for tests or traces that cover the side-effect axis, not only the dependency that the repair removed.

Coverage and filler repair

  • Returned findingThe agent cannot read the loop as zero or nonzero because contract, test, runtime, or policy filler evidence is missing.
  • Why ArchSig sees itHomotopy and coverage readings keep unfilled loops, missing filler evidence, coverage gaps, and measured boundaries visible instead of smoothing them into a score.
  • Review actionRepair the ArchMap first: add measured source refs, tests, traces, docs, or explicit unavailable/private/out-of-scope gaps.
  • Confirmation checkRerun the Skills and compare architectural holes, nonzero holonomy loops, coverage gaps, and the action queue.
