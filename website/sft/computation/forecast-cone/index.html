ForecastCone

A ForecastCone is the set of bounded future paths reachable from a field estimate under selected support, policy, and horizon. It is a reachability object, not a single prediction.

Question

What futures remain reachable if we fix the field model, support relation, policy, and horizon?

Cone semantics

The cone begins at a selected field estimate. At each step, it follows operations supported by the field and accepted by the selected policy. The horizon bounds how far the computation is allowed to unfold.

Definition 2.11. A ForecastCone is a bounded set of reachable field paths relative to a start field, support family, policy assumptions, and horizon.

SFT statusformal targetSource: SFT section 12; website page does not add Lean definitions.

After artifact action

A PRD, spec, issue, or AI proposal may change support before any code is changed. SFT can therefore compare cone families after an artifact action: which future path families become newly supported, which become harder, and which unknowns remain.

This comparison is useful precisely because it is not a point prediction. It tells reviewers what the artifact makes reachable under the selected model.

Boundary

Non-conclusion 2.12. A ForecastCone does not claim probability weighting, forecast correctness, causal proof, global safety, or that a team will actually choose one of its paths.
