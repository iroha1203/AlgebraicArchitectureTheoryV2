Question

Which questions are computational, and which remain research or calibration work?

Problem map

  • Field ReconstructionBuild a trace-grounded SoftwareFieldEstimate with missing evidence preserved.
  • Operation Support InferenceEstimate which operation families the field makes available, cheap, costly, selected, or blocked.
  • ConsequenceEnvelope GenerationProject bounded cone evidence into a reviewer-readable report.
  • Cone NarrowingCompare reachable path families under changed support or policy constraints.
  • Support Safety CheckingCheck selected preservation or simulation properties under explicit assumptions.
  • Feedback UpdateUse observed review, CI, PR, incident, or outcome refs to update field estimates.
  • AI Proposal GovernanceEvaluate proposal pressure, shortcut witnesses, review feedback, and allowed support boundaries.
  • Lifecycle DecisionCompare repair, migration, contraction, deletion, and end-of-life decisions as field capacity changes.

Claim maturity

These problems do not all have the same status. Some are formal targets, some are tooling artifacts, and some require empirical calibration. The website keeps those levels separate so a command output is not mistaken for a theorem or for a calibrated empirical claim.

Definition 2.19. An SFT computational problem is a bounded task over selected field evidence whose outputs must state theorem status, tooling status, empirical status, and non-conclusions separately.

Boundary

Non-conclusion 2.20. Listing a computational problem does not mean the current tool solves it completely. Implemented artifacts, formal targets, and empirical research gaps remain distinct.
