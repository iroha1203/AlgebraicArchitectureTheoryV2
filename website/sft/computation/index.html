Introduction

The computational move in SFT is not to shrink the theory until it becomes a static metric. It is to keep the large object in view, then choose a bounded slice that can be inspected. The field remains rich; the computation states exactly which part of that richness has been selected.

A SoftwareFieldEstimate records selected evidence, unavailable evidence, and unknown remainder. AAT then supplies the local algebra for the architecture projection. ArchSig supplies bounded observations. SFT adds support, policy, step semantics, horizons, path sets, envelope projections, and feedback hooks.

This is how "software evolution" becomes a computable object. We do not compute the future as a point. We compute a bounded space of paths: which architecture futures remain reachable when this field, this support relation, this policy, this observation boundary, and this horizon are selected.
