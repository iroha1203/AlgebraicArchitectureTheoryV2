ArchSig-SFT Pipeline

The ArchSig-SFT pipeline connects Markdown PRD, Spec, Issue, and AI Proposal artifacts to bounded forecast artifacts. It is a report-generation surface, not a theorem generator.

Question

What does the implemented tooling surface produce, and what does it deliberately not conclude?

Artifact chain

The pipeline keeps artifact refs, support refs, horizon, measurement boundary, theorem precondition status, unknown remainder, and non-conclusions visible. That shape makes it useful for review and later calibration because the report can be joined with observed PR, review, CI, incident, and outcome references.

Definition 2.21. The ArchSig-SFT pipeline is the implemented tooling surface that projects selected artifact input into bounded forecast and review artifacts while preserving evidence boundaries.

Claim boundary

The pipeline output can say: under this artifact, this support estimate, these selected axes, and this horizon, these path classes and affected boundaries should be reviewed. It cannot say: this future will happen, this theorem is proved, this organization is safe, or this forecast has been calibrated.

For command and artifact details, read ArchSig commands and ArchSig artifacts.

Boundary

Non-conclusion 2.22. ForecastConeSkeleton and ConsequenceEnvelope artifacts do not certify a Lean theorem, forecast correctness, causal effect, architecture lawfulness, or global safety.
