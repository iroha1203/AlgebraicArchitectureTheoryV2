How should a field govern the operation support generated by AI agents?

Field participant

An AI coding agent is an artifact interpreter, proposal generator, local pattern amplifier, shortcut generator, review load shifter, and field participant. Its behavior depends on the codebase and artifacts that surround it.

Vocabulary

AI agent as field participant

A proposal generator whose output is shaped by field memory, visible patterns, artifacts, prompt policy, and governance.

Amplifies
Local patterns, default paths, hidden assumptions, and review load shifts.
Requires
Bounded support, shortcut witnesses, mediation records, and posterior field update.

Bounded support

SFT does not try to prove general AI safety. It asks a more precise engineering question: under this field estimate, which operation families are allowed, which are forbidden, which are unknown, and which theorem or forecast boundaries should reviewers see?

Vocabulary

Allowed operation support

The bounded candidate operation family an AI proposal may explore under prompt, policy, theorem, review, and field boundaries.

Contains
Allowed operation families, forbidden support, theorem boundaries, review requirements, and unknown remainder.
Does not prove
General AI safety, model reliability, or architecture lawfulness.

Shortcut witnesses

Shortcut witnesses name the paths that look cheap to the agent but threaten the selected architecture field: forbidden edges, missing invariants, runtime boundary crossings, theorem boundary gaps, or review load transfers.

Vocabulary

Shortcut witness

Evidence that a low-cost generated path crosses a forbidden edge, misses an invariant, hides runtime risk, or shifts work into review.

Reports
Forbidden edge, missing invariant, runtime boundary, theorem boundary, or review load.
Feeds
Review mediation, CI instrumentation, issue decomposition, and posterior field update.

Proposal mediation

AI governance is not only pre-generation policy. It is proposal accounting after generation. Accepted, rejected, delayed, reworked, unresolved, and incident-linked proposals all become evidence about the field.

Mediation record Field meaning
Accepted The generated operation became part of the realized field path.
Rejected The field exposed unsupported or unsafe proposal pressure.
Delayed The proposal found a path whose support needs design, ownership, or missing invariant work.
Reworked Governance redirected the proposal into a different supported path.
Incident-linked The proposal becomes runtime feedback and shortcut evidence.

Feedback

Definition 3-2. AI proposal governance is the bounded field intervention that records allowed support, forbidden or shortcut paths, theorem boundary items, review outcomes, and CI feedback for AI-mediated software evolution.

Proposition 3-2. AI governance is field governance when it constrains generated proposal support and records the observed mediation outcome for later field updates.

Proof idea. The AI proposal stream supplies candidate operations. Prompt and policy boundaries restrict the candidate support; shortcut witnesses classify dangerous low-cost paths; review and CI mediate the transition; observed outcomes are stored in the posterior field. These are exactly support, policy, observation, mediation, and feedback transformations.

Case 3-2. If an agent patches PaymentAdapter for a coupon feature because local examples make it easy, SFT does not read the event only as model failure. It reads the field: the cheap path was visible, the invariant was hidden, and governance did not yet make the lawful path easier.
