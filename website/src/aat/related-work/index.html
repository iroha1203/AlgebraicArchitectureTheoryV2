Novelty claim

AAT's distinctive contribution is the combination of atom root, law-relative obstruction circuit, multi-axis architecture signature, operation calculus, path geometry, analytic representation, and Lean status discipline. The combination turns architecture reasoning into claims whose evidence, scope, and formal status can be inspected.

Novelty 1.1. The central novelty is compositional: AAT joins primitive architectural facts, finite presentations, selected law universes, law-relative minimal bad molecules, multi-axis signatures, operation packages, path geometry, and formal status into one reusable architecture reading discipline.

Contribution

AAT organizes neighboring architecture methods into a single claim grammar. The grammar asks which atom family is selected, which object is formed, which law family is active, which obstruction witness is minimal, which signature axes carry the reading, and which operation package licenses a change.

Architecture description

Architecture description standards provide the reader-facing vocabulary of viewpoints and concerns. AAT gives that vocabulary theorem-shaped carriers: a selected presentation, an architecture object, laws over that object, and a boundary around the claim.

Position 3.1. A viewpoint can be read as a selected projection; a concern can be read as a law family; a correspondence rule can be read as an invariant or diagram that a theorem package may preserve.

Description concept AAT carrier Reading
View SelectedPresentation The finite slice where a judgement is made.
Concern DesignLaw / invariant family The law package that gives the concern a theorem shape.
Correspondence Diagram / bridge theorem The relation whose commutation, preservation, or reflection is checked.

Review methods

Review methods such as ATAM make risk concrete through scenarios and tradeoffs. AAT preserves that concreteness by turning a scenario into a witness family and a signature axis. The result is a review claim that names its law universe and its evidence.

Position 4.1. A scenario is an input to AAT when it selects an operation path, an invariant, a witness rule, or a diagram whose filling status can be inspected.

Position 4.2. A review finding becomes theorem-shaped when it can name its selected object, law universe, witness family, measurement axis, and status label. The review still supplies judgement and prioritization; AAT supplies the claim grammar.

Metrics and graphs

Metrics and graph analysis become computable surfaces inside AAT. Cycle count, violation count, nilpotence, spectral radius, curvature value, and aggregate score all belong to a selected valuation or representation. Their proof strength depends on the preserving and reflecting theorems attached to that surface.

Position 5.1. The same dependency graph can support a graph reading, matrix reading, signature reading, and obstruction reading. AAT keeps those readings aligned by making their bridges explicit.

Position 5.2. Metrics become strongest when they are read as valuations: nonzero values support selected obstruction readings, measured zero supports selected absence only through zero-reflection assumptions, and aggregate values remain tied to their selected axes.

Formal methods

Lean gives AAT a checked core for definitions, bridges, equivalences, finite examples, and counterexamples. The formal subset is deliberately stated through status labels: proved, defined only, future proof obligation, and empirical hypothesis.

Position 6.1. AAT's formal contribution is strongest where a theorem package connects a selected architecture structure to a selected signature, lawfulness, zero-curvature, path, representation, or counterexample reading.

Formal-methods pattern AAT use
Definition Atom, molecule, law, witness, signature axis, operation, path, representation.
Soundness theorem Selected witness, repair, synthesis, path, or representation conclusion.
Counterexample theorem Static/semantic separation, endpoint-only trajectory reading, design-pattern overreading.

Category models

Categorical vocabulary is most useful in AAT when it is tied to concrete architectural evidence: reachability as a thin category, paths as endpoint-indexed sequences, homotopy as a generated relation, and diagram filling as selected proof of commutation.

Position 7.1. The categorical reading remains architecture-facing: it explains when two change paths can be identified, which square can be filled, and which selected observation is preserved.

Position 7.2. Thin reachability, endpoint-indexed paths, generated homotopy, and diagram filling give four different levels of structure. AAT keeps the levels separate so path count, route identity, observation equality, and filler evidence each have the right carrier.

Reading rules

Adjacent method gives AAT asks AAT produces
A view or model Which selected presentation and object? A bounded architecture judgement.
A scenario or risk Which witness family and law axis? A signature reading and theorem boundary.
A metric or graph fact Which preservation or reflection theorem? A computable representation claim.
A formal specification Which architecture package does it discharge? A Lean status label and formal anchor.

This is the reason AAT reads as a synthesis rather than a replacement. The neighboring method supplies the surface; AAT supplies the architecture-specific package that makes the claim inspectable.
