The rules of the instrument

What ArchSig says — and what it refuses to say.

ArchSig's usefulness comes from a small number of hard rules: observation never contains judgment, verdicts come from a closed five-word vocabulary, every conclusion is relative to declared evidence, and what cannot be measured is recorded as silence — not as failure, and not as safety.

Three layers, three responsibilities

Every ArchSig run is the same computation: ArchMap × (LawPolicy + MeasurementProfile) → measurement. The three inputs are deliberately separate layers, because when a conclusion turns out wrong you need to know which layer was wrong — the observation, the selection, or the computation.

  • ArchMap observesA source-grounded record of what is the case: components, relations, declared section values, contexts, and the selected cover, each backed by source refs. It contains no evaluation of any kind.
  • LawPolicy + MeasurementProfile selectWhich evaluators run, over which scope, at which severity, against which basis documents — and the concrete finite regime: the cover, the coefficient, the witness family, the bounds. Selection is an institutional act, and it is recorded as one.
  • ArchSig computesThe CLI turns evidence plus selection into bounded verdicts and invariants, deterministically. It never adds evidence, never extends scope, and never softens a verdict.

Because the layers are separate, a bad reading has a findable owner: a wrong observation is fixed in the ArchMap, a wrong selection in the policy, a wrong computation in ArchSig itself. Mixing the layers would make that attribution impossible — which is why the validators reject inputs that try.

Observation is not judgment

An ArchMap may say "the payment adapter derives the discount per line, rounded half-to-even". It may never say "the payment adapter's rounding is wrong". The words violation, mismatch, bad, nonzero do not belong to observation; conclusions exist only as computation output. The ArchMap validator treats judgment vocabulary inside observations as a hard error.

One nuance matters: the semantic reading inside an observation — what a piece of code means, which convention it implements — is legitimately a human (or agent) judgment call about meaning, and ArchSig protects it. The same function can mean different things in different bounded contexts, and only a reader can say which. What is banned is evaluation, not interpretation. Interpretation is the observer's job; evaluation is the instrument's.

In the one-cent drift, each context declares the money convention its code actually implements, grounded in the symbols that carry it. No context is "the wrong one" — the measured obstruction is a property of the loop, computed later, by the evaluator the policy selected.

The five-valued verdict

Every structural verdict row carries exactly one of five values. The discipline is that no value is ever rounded into a neighbor — an unmeasured row does not become "probably fine", and a blocked computation does not become "error".

VerdictIt meansIt does not mean
measured_zeroThe evaluator ran on the declared evidence and the obstruction is absent — a positive result with a certificate."Nothing was found because nothing was looked at."
measured_nonzeroThe evaluator ran and the obstruction is present, with concrete support named in the packet.A style warning, a score, or a proof that the code misbehaves at runtime.
unmeasuredThe selected evidence does not cover this row.Safe. It is not measured_zero and is never merged with it.
unknownEvidence exists but does not determine the answer under this profile.An error, or a promise that more computation would decide it.
not_computedThe evaluator did not run — for example ag.saga-descent without a repair plan stays not_computed with a silence_by_design boundary.A failure. Some silences are the designed behavior.

A sixth word appears at the gate: violated_assumption_dependency, for rows whose declared assumptions were violated. A gate policy must map all six explicitly, using only three actions — pass, pass_with_boundary, block — and the four non-terminal values (unmeasured, unknown, not_computed, violated_assumption_dependency) may not map to a plain pass. Rounding silence into passing is structurally impossible to write into a valid policy.

Everything is profile-relative

Conclusion codes end in UNDER_PROFILE for a reason. MEASURED_H1_OBSTRUCTION_UNDER_PROFILE asserts: over the selected cover, with the selected coefficient, on the declared section values, the mismatch class is nonzero. It does not assert that the codebase is broken everywhere, that production will fail, or that no other defect exists. Conversely the zero is a real zero of that measurement — conventions were observed everywhere in the cover and they glue.

This relativity is not a hedge; it is what makes the result checkable. Every run's packet carries the input digests, the assumption ledger (which premises were checked, which remain assumed, and what checked them), and the exact evidence refs. A reviewer who disputes a conclusion disputes something concrete: the cover, a section value, an assumption's status.

Comparability: when two runs may be compared

archsig compare juxtaposes two runs at record level, and it grades how comparable they are before saying anything about transitions:

  • identicalSame ArchMap digest, LawPolicy digest, profile fingerprint, and tool version.
  • verdict-rowSame profile fingerprint, site cover digest, and tool version — verdict rows can be matched one-to-one even though the evidence changed. This is the normal PR case.
  • not-comparableThe site itself changed — for example the PR added a context, as the one-cent drift head does. The report records both independent conclusions and a typed boundary, and refuses row-level claims.

Compare never claims that an obstruction was "introduced by" or "fixed by" a change, never transports a cohomology class across runs, and never decides causality. Those readings are left to the humans and agents looking at the juxtaposition — with the gate acting on each run's own verdicts, which is why a not-comparable head run can still be blocked on its absolute rule.

Silence as a feature

ArchSig speaks only within the vocabulary its inputs select, and is silent about everything else. Silence appears in the artifacts as typed boundary statements — this profile did not observe X, this evaluator needs a repair plan to run — placed near the conclusions they bound, and never as a to-do list of everything the tool did not check. A long list of non-conclusions would itself be a claim ("these are the things worth checking"), and ArchSig has no evidence for it.

Practically: when reading output, treat boundaryStatements as the precise edge of what was measured, and treat anything outside it as simply unaddressed — not risky, not safe, unaddressed.
