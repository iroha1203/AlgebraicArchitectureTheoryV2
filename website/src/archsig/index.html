ArchSig v0.5.3

Architecture measurement for fast-moving code

Every module is correct. The system still overcharges by one cent.

ArchSig measures the one thing file-by-file review cannot see: whether a set of individually correct modules still glues into a consistent whole. It reads declared evidence, computes a bounded verdict, and blocks the PR in CI with the exact modules that carry the mismatch.

The one-cent drift

A pull request lands in a realistic Rust commerce service. Every unit test is green in every build configuration. Every hunk of the diff is locally justified and defensible in review. And the customer's card gets charged one cent more than the checkout page displayed.

Three modules handle money under three different conventions — each one exactly what its owner's spec demands. The defect is not in any diff hunk; it lives on the loop checkout → payment → settlement → checkout, which no single file view contains. With three conventions on a cycle, no per-module fix reconciles the loop: the disagreement is global. That distinction — repairable locally versus obstructed globally — is exactly what ArchSig measures, and it is invisible to pairwise interface checks.

You can reproduce this in about fifteen minutes with one script — see Getting Started. Every conclusion above is a real, deterministic measurement over declared evidence, not a heuristic score.

Three tools, one contract

The tool layer has three components with one shared discipline: measurement first, and no component speaks beyond its inputs.

  • ArchSig measuresA Rust CLI that reads declared architecture evidence (ArchMap), a selected policy bundle (LawPolicy + law-equation-surface + MeasurementProfile), and computes bounded structural verdicts. analyze is the primary workflow; compare and gate are the PR / CI decision surfaces.
  • ArchView projectsA no-build 3D viewer that renders the measured geometry — patches, overlaps, and the exact seams that fail to glue. It adds no new verdict; it shows what ArchSig measured. Open the live demo.
  • FieldSig previews evolutionAn evolution-measurement layer that reads ArchSig measurement packets as bounded input for software-evolution analysis. It is not part of this manual; it grows with SFT.

Five Skills: the agent operates, you supervise

ArchSig is Skill-first: the primary operator is an LLM agent, and the release bundle ships the Skill documents the agent follows. You do not need to learn the input schemas — you ask for an analysis, approve the scope, and read the returned findings.

  • archmap-createrSurveys your repository in two independent passes and builds a source-grounded archmap/v0.5.3 evidence file, with a scope manifest you approve first.
  • law-policy-createrSelects evaluators from the registry, authors law-policy/v0.5.3, law-equation-surface/v0.5.3, and measurement-profile/v0.5.3, then fixes them with an archsig-policy-bundle/v0.5.3. If evidence does not justify a selector, it asks instead of guessing.
  • archsig-readerRuns analyze, reads every output artifact, checks the cited source, and returns bounded review findings.
  • archsig-pr-reviewerRuns base/head measurement, compare, and gate, then turns the result into source-first review questions for the PR.
  • repair-plan-createrAuthors archsig-repair-plan/v0.5.3 artifacts for SAGA descent runs — the measured path from "blocked" to "repaired".

What you get back

The output is a small set of JSON artifacts designed to be read — by the agent first, then by you. The headline is always a positive, bounded conclusion, never a vague score.

  • A conclusion you can act onMEASURED_H1_OBSTRUCTION_UNDER_PROFILE means "a cross-context glue mismatch was measured under the selected profile". NO_MEASURED_H1_OBSTRUCTION_UNDER_PROFILE is a real measured zero, not an absence of data. When SAGA repair data is supplied and validated, the vocabulary sharpens further: MEASURED_NONGLUING_RESIDUAL_CLASS and REPAIR_GLUES_WITHIN_SELECTED_COMPLEX speak at the level of the residual class itself.
  • The exact supportThe measurement packet names the overlap edges and evidence atoms that carry the mismatch, with references back to source paths and symbols.
  • A CI decisiongate maps the five-valued verdict to pass / pass_with_boundary / block under a policy your CI owns — and exits nonzero on block.
  • Declared silenceWhat the selected evidence cannot support is recorded as unmeasured or not_computed with a boundary statement — never rounded up to "safe" or down to "error".

How to read each artifact, field by field, is in the Manual. What each verdict word is allowed to mean is in Concepts.

Install

Download the release bundle for your platform, or build from source. The bundle ships the CLI, the five Skills, the command docs, and ArchView.

  • GitHub ReleasesPrebuilt archives for Linux (x86_64, arm64), macOS (universal), and Windows, with SHA256SUMS.txt.
  • Build from sourcecargo build --release --manifest-path tools/archsig/Cargo.toml in the repository checkout. The demo and fixtures live in the same repo.
