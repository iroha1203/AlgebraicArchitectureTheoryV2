Architecture as a Paper Atlas

Walk from a codebase overview to the Atom and source location that matters.

ArchView reads ArchMap directly and lays its supplied Covers, Contexts, Subjects, and Atoms onto a quiet three-dimensional atlas. Optional ArchSig results appear on the same geometry, so architecture understanding and analysis share one place, camera, and selection.

Live atlas: the train-ticket codebase

The committed demonstration contains the complete measured map: 2,118 Atoms, 43 Contexts, 3 Covers, and 440 source records. Context plates appear first; Atom glyphs arrive progressively without sampling the input.

  • Open the full Paper AtlasSearch, select, zoom, switch to the keyboard outline, or load another ArchMap. Three.js is loaded from its CDN.

How to read the Paper Atlas

  • CoverThe chosen finite family sets the atlas overview.
  • Context platesEach plate comes from an explicit Context and its Atom membership.
  • Subject groups and Atom glyphsShape records Atom kind; grouping records the supplied subject within its Context.
  • Restrictions and shared supportConnectors and translucent supports come only from explicit restriction and membership facts.
  • Source landingThe drawer displays the supplied file, symbol, and line metadata. ArchView does not load source bodies.

The 3D Atlas and Outline / Table share the same Cover → Context → Subject → Atom → Source selection. Architecture, Analysis, and Improve modes keep that selection and camera while changing the information shown.

Use your ArchMap

Choose Load ArchMap in the top bar, or serve the release bundle and pass a same-origin file with ?archmap=./your-map.json. Choose Load ArchSig run to add a compatible run directory as an analysis overlay.

Rotate by dragging, pan with right-drag or Shift-drag, and zoom with the wheel. The toolbar supplies overview, focus, top, isometric, front, and reset views. The Outline / Table provides the complete keyboard path.
