Abstract

Algebraic Architecture Theory formalizes software architecture through objects, operations, invariant families, obstruction witnesses, signature axes, and theorem boundaries. It gives architecture review a mathematical grammar: design principles become operations on invariants, failures become explicit witnesses, and quality becomes a multi-axis signature.

Claims in AAT carry their scope explicitly: the selected universe, observation context, coverage boundary, and exactness assumptions are part of the statement. This makes architectural reasoning composable: a review can state which invariant is preserved, which obstruction is witnessed, which axes are measured, and which theorem package licenses the conclusion.
