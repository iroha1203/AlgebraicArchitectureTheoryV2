Novelty boundary

AAT is built from familiar neighboring tools, but it organizes them around a specific architecture question: which operation preserves which invariant, under which evidence boundary, and with which obstruction signature?

Positioning A.1. AAT's novelty is the bounded claim grammar that ties an architecture operation to invariant families, obstruction witnesses, signature axes, theorem boundaries, and explicit scope. The neighboring methods supply inputs, representations, or proof techniques; AAT supplies the architecture-specific package that reads them together.

Contribution pattern

How to read an adjacent-work comparison
Question AAT reading Contribution
What does the neighbor supply? A notation, review process, metric, graph fact, model, or proof technique. AAT keeps the source concept visible and imports it with an explicit role.
What does AAT import? A selected evidence boundary, law family, witness universe, axis, or representation. The import remains relative to the selected universe and assumptions.
What does AAT add? The theorem boundary, obstruction reading, signature status, and explicit scope. The result is a citable architecture claim package.

Positioning A.2. AAT is a boundary discipline for architecture claims. It uses architecture descriptions, ATAM-style scenarios, metrics, graph analyses, theorem proving, and categorical vocabulary as inputs for bounded invariant, witness, signature, and theorem-boundary packages.

ISO 42010

ISO/IEC/IEEE 42010 is the natural neighbor for architecture description: stakeholders, concerns, viewpoints, views, and model kinds organize how an architecture is documented. AAT takes that boundary discipline and turns it into a reasoning surface for selected architecture claims.

AAT asks a different question. Given a selected architecture object and observation boundary, which invariant family is required, which witness universe can expose a violation, which signature coordinates were measured, and which theorem boundary licenses the result?

Neighbor concept AAT reading AAT contribution
Viewpoint / view Selected observation context and evidence boundary. A theorem or report states the invariant reading for that selected view.
Model kind Carrier, graph, diagram, metric, or report shape used by a theorem or tool claim. Different model kinds require explicit bridge claims.
Concern Invariant family, required law, or signature axis. The concern becomes measurable when paired with an axis and evidence boundary.

ATAM

ATAM and related scenario-based methods organize quality attribute reasoning around scenarios, risks, sensitivity points, and tradeoffs. AAT supplies a formal claim discipline for the parts of a review that can be stated as bounded laws, witnesses, signature axes, and theorem packages.

  • Quality attribute scenario AAT can read a scenario as a selected observation context plus required invariant family when the evidence is precise enough.
  • Risk and tradeoff AAT represents selected risks as obstruction witnesses or nonzero signature axes; tradeoffs become axis-local movements across a signature.
  • Review conclusion AAT attaches a theorem boundary or artifact boundary to the conclusion.

This makes AAT complementary to review. A review can identify a risk, sensitivity point, or tradeoff; AAT asks whether that risk has a selected witness, whether the corresponding signature axis was measured, and whether a theorem package or report boundary licenses the proposed conclusion. The result is a review finding that can be read as a bounded architecture claim.

Architecture metrics

Architecture metrics often summarize dependency, coupling, complexity, volatility, or risk. ArchitectureSignature reads quality as a multi-axis diagnostic whose coordinates carry measurement status, evidence, universe, and scope.

Signature boundary A.3. Signature comparison is meaningful on selected axes where both sides are measured and an axis-local order has been declared. The signature keeps static split, semantic commutativity, runtime exposure, projection exactness, and empirical outcomes available as separate coordinates.

Graph analysis

AAT reuses graph-level dependency analysis for selected static and runtime relations: reachability, layering, acyclicity, paths, adjacency-style representations, and finite metric calculations. The distinctive move is the bridge from graph fact to architecture claim: a selected graph fact becomes evidence for a named invariant, witness, or signature axis.

  • Reused Directed dependency graphs, finite component universes, reachability, witness counts, path and matrix-style representations.
  • Added by AAT Law families, obstruction witnesses, signature status, and theorem boundaries that state which graph fact supports which architecture claim.
  • Result Graph analysis becomes theorem-bound evidence for selected architecture readings.

Formal methods

Formal methods, model checking, type systems, theorem proving, and refinement techniques provide ways to verify precise specifications. AAT uses theorem proving as an anchor for selected architecture statements and connects checked declarations to public theorem boundaries.

The current Lean role is inspectable formal anchoring: selected finite static lawfulness, selected required witnesses, and selected required signature axes under explicit coverage and exactness assumptions. Runtime, semantic, and empirical readings use separate packages when those axes enter the claim.

Formal anchor A.4. The finite static structural core gives AAT a citable formal anchor for selected lawfulness, required signature axes, finite universe, coverage, and exactness assumptions. Runtime, semantic, and empirical readings are organized separately.

Category-theoretic models

AAT uses categorical vocabulary where it clarifies structure, composition, projection, lifting, and obstruction. Its current ComponentCategory is intentionally thin: it forgets path counts and walk lengths.

Quantitative path information belongs to Walk, Path, adjacency matrices, finite metric representations, or future free-category constructions. This separation keeps categorical structure from swallowing measurement semantics.

Category vocabulary boundary
Concept Current AAT use AAT separation
ComponentCategory Thin reachability category over selected components. Path count, walk length, and weighted propagation live in metric or path representations.
Projection / lifting Explicit bridge between concrete and abstract architecture objects or operations. Exactness is supplied by the selected bridge package.
Analytic representation Structure-preserving and reflecting directions under coverage and completeness premises. Semantic completeness is stated as a separate premise or theorem.

Novelty claim

AAT's novelty is the bounded architecture-reasoning discipline that packages architecture views, scenario review, dependency graphs, software metrics, theorem proving, and category theory into one claim grammar. In that grammar, design principles are operations that preserve or improve invariant families, failures are explicit obstruction witnesses, and quality is read as a multi-axis signature under theorem boundaries.

Novelty boundary A.5. AAT contributes an architecture-specific packaging of operation, invariant, obstruction witness, signature axis, and theorem boundary. Its novelty is the disciplined composition of these pieces into a reusable reading discipline for software architecture.

Area What AAT reuses What AAT contributes
Architecture description Views, model boundaries, and concern separation. Theorem boundary, witness universe, and signature status attached to claims.
Architecture evaluation Scenario, risk, and tradeoff vocabulary. Bounded law and obstruction readings for review conclusions.
Metrics Finite measurements and axis-specific readings. Multi-axis ArchitectureSignature with measured, unmeasured, out-of-scope, and scoped readings.
Graphs and formal methods Dependency graphs, reachability, theorem proving, and bridge lemmas. Architecture-specific packaging of invariant preservation, witnesses, and proof boundaries.

Research program A.6. AAT turns adjacent architecture methods into a common algebraic reading surface. Description supplies selected views, review supplies scenarios and risks, metrics supply measured axes, graph analysis supplies finite structural evidence, formal methods supply checked anchors, and categorical models supply composition and lifting vocabulary.
