Design Principle Layers

Design principles become precise in AAT when they are read as bounded law packages. SOLID, Layered Architecture, Event Sourcing, Saga, Circuit Breaker, and Replicated Log live on different layers because they preserve different invariants under different evidence boundaries.

Research question

What does a design principle actually preserve, and at which architectural layer is that preservation claimed?

AAT does not treat a principle name as a theorem. "SOLID", "Clean Architecture", "Event Sourcing", "Saga", and "Circuit Breaker" are not magic labels. Each becomes mathematical only after selecting a state space, an operation family, an invariant family, evidence boundaries, and non-conclusions. The layer tells the reader which preservation problem is being solved.

Law package

A design principle layer is a family of bounded law packages. A package chooses the objects it talks about, the operations that can change those objects, the invariants to preserve, and the theorem boundary under which preservation is valid.

Definition 1.1. A design principle layer is a bounded family of law packages that share a state space, operation vocabulary, invariant vocabulary, and evidence boundary.

Proposition 1.2. A law package is composable only through the invariants and operations it explicitly selects. A principle name alone does not supply a cross-layer theorem.

Proof idea. The generic closure relation is operation-invariant preservation: operations in \(T\) preserve invariants in \(S\). Lean states this as T subset Ops(S) iff S subset Inv(T). Nothing in that closure says that an invariant from another layer is also preserved.

Layer map

The same architecture may need several law packages at once. A component replacement may be locally substitutable, a module graph may be globally layered, a payment workflow may satisfy selected replay and compensation laws, and a runtime dependency may be protected by a circuit breaker. These are related readings, not the same theorem.

Layer Principle families Selected invariant shape
Local contract SOLID, SRP, ISP, LSP, DIP, local replacement Projection soundness, observation factorization, selected substitution, responsibility cohesion.
Global structure Layered Architecture, Clean Architecture Strict layering, decomposability, boundary-policy soundness, abstraction-policy soundness.
State-transition algebra Event Sourcing, Saga, CRUD when given a selected carrier Replay lawfulness, projection lawfulness, roundtrip laws, weak compensation.
Runtime dependency Circuit Breaker, isolate, protect Path localization, interaction protection, bounded runtime flatness.
Distributed convergence Replicated Log, quorum protocols Quorum evidence, ordering condition, conditional convergence for selected entries.

Non-conclusion 2.1. A layer assignment is not a ranking of principles. It is a scope marker: each layer states which invariants are selected and which conclusions remain outside the package.

Local contract layer

SOLID lives primarily at the local contract layer. SRP selects a responsibility boundary and local cohesion condition. ISP refines the projection visible to a client context. LSP states selected observation equivalence under replacement. DIP records how concrete dependency evidence travels through abstraction. OCP is read as extension by operations that preserve the selected local contract invariants.

Proposition 3.1. A proof-carrying local replacement operation preserves projection soundness, LSP compatibility, and DIP compatibility at its target.

Proof. The Lean structure LocalReplacementOperation carries a target-side LocalReplacementContract. Theorems localReplacementOperation_preserves_projectionSound, localReplacementOperation_preserves_lspCompatible, and localReplacementOperation_preserves_dipCompatible extract the selected local invariants from that contract.

Proposition 3.2. SRP and ISP are formalized as bounded packages, not natural language judgements.

Proof idea. SRP uses ResponsibilityBoundary.Total, ResponsibilityBoundary.Functional, and ResponsibilityBoundary.EdgeCohesive. ISP uses target projection soundness and observation factorization. Both record natural-language optimality and organizational quality as non-conclusions.

Global structure layer

Layered Architecture and Clean Architecture are global structure packages. Their invariants talk about the dependency graph as a whole: whether a strict layer assignment exists, whether the graph is decomposable, and whether boundary and abstraction policies are respected.

Proposition 4.1. Edge-subset structural restrictions preserve strict layering, decomposability, boundary-policy soundness, and abstraction-policy soundness.

Proof. StructuralRestrictionOperation carries an EdgeSubset target.G source.G. The preservation theorems apply existing edge-subset results for strict layering and decomposability, and use the same edge-subset witness to transport boundary and abstraction policy soundness.

Non-conclusion 4.2. A global structural package does not classify practical layer naming conventions, prove runtime or semantic decomposability, or prove global flatness preservation unless those are supplied by further packages.

State-transition layer

Event Sourcing and Saga are state-transition algebra packages. They do not primarily assert that a dependency graph is layered. They assert laws about replay, projection, roundtrip, and compensation over selected finite cases and a selected semantics.

Proposition 5.1. Event Sourcing is represented by event-sequence monoid laws plus selected replay and projection lawfulness.

Proof. Lean proves eventSequenceMonoidLawful for finite event histories and defines EventSourcingLawFamilyLawful as selected replay and projection lawfulness. The design-pattern closure theorem then records those selected invariants.

Proposition 5.2. Saga is represented as selected weak recovery, not as a general inverse law.

Proof. SagaWeakRecoveryLawful is defined through selected compensation cases. SagaNonConclusionClause records that compensation is not claimed to be a general inverse, and that all failure-mode coverage and operational recovery-cost improvement remain outside the theorem package.

Non-conclusion 5.3. CRUD is not automatically a design-pattern theorem. It becomes part of this layer only after a selected state carrier, operation family, invariant family, and evidence boundary are supplied.

Runtime and distributed layers

Circuit Breaker, isolate, protect, and Replicated Log live beyond static structure. Runtime protection packages are bounded by a component universe and selected region. Distributed convergence packages are bounded by a failure model, quorum predicate, ordering relation, convergence predicate, and selected entries.

Proposition 6.1. Runtime protection operations preserve selected path localization, runtime interaction protection, and bounded runtime flatness only under the assumptions of the relevant isolate or protect law package.

Proof. The runtime package cases carry isolateRuntimeLocalizationLaw.AssumptionsHold or protectRuntimeProtectionLaw.AssumptionsHold. The preservation theorems call the corresponding law-package conclusion accessors.

Proposition 6.2. Replicated Log lawfulness is conditional convergence over selected entries, not proof of an arbitrary consensus implementation.

Proof. ReplicatedLogLawFamilyLawful bundles quorum, ordering, and convergence conditions for selected entries. ReplicatedLogNonConclusionClause records that unconditional availability, partition tolerance, unconditional convergence, protocol completeness, and concrete implementation correctness are outside the package.

Inter-layer non-implications

The main value of layering is negative as well as positive. It prevents a theorem from being read beyond its layer. A local LSP theorem does not prove structural decomposability. A strict dependency layer does not prove replay correctness. A circuit breaker package does not prove semantic compensation. A quorum package does not prove API usability.

Counterexample 7.1. A local replacement can preserve selected LSP and DIP compatibility while the abstract graph remains cyclic. Local contract compatibility does not imply global structural decomposition.

Counterexample 7.2. A strict layered graph can still replay event histories incorrectly. Static layer soundness does not imply Event Sourcing replay lawfulness.

Counterexample 7.3. A selected Saga compensation can be lawful for listed cases while an unlisted failure mode remains uncovered. Weak recovery is not failure-mode completeness.

Counterexample 7.4. A replicated-log package can provide conditional convergence for selected entries without proving unconditional availability or a concrete protocol implementation.

Worked example

Consider a payment capture service that is being split into a PaymentPort, a GatewayAdapter, a Ledger, and an asynchronous receipt workflow. AAT reads this as several law packages, not one design slogan.

The page-level conclusion is precise: a review can ask which package failed. If the adapter violates a boundary edge, the structural package is nonzero. If retry can double-charge, the state-transition or runtime package is nonzero. If a client sees different selected observations after replacement, the local contract package is nonzero. These failures may interact, but AAT does not identify them without an explicit bridge.

Formal reading

The Lean surface for this chapter is spread across the Formal/Arch/Patterns modules and the generic operation-invariant closure. The modules define bounded design-pattern schemas, preservation theorems, closure laws, and recorded non-conclusions for each layer.

Formal status 9.1. The local contract, SRP, ISP, structural, state-transition, Event Sourcing, Saga, runtime protection, and replicated-log design pattern schemas are defined in Lean.

Formal status 9.2. Each pattern exposes closure-law theorems through the generic DesignPattern schema and records explicit non-conclusions.

Formal status 9.3. Cross-layer theorem identification is not asserted. Layer boundaries remain part of the claim discipline unless a separate bridge theorem is supplied.

Chapter summary

Design principles become AAT objects when they are packaged as selected invariant-preservation laws. The package records what is preserved, which operations preserve it, which evidence supports the claim, and which conclusions are deliberately excluded.

This is why the layers matter. SOLID belongs mainly to local contracts; Layered and Clean Architecture belong to global structure; Event Sourcing, Saga, and CRUD require state-transition carriers; Circuit Breaker and Replicated Log require runtime and distributed evidence. AAT keeps these claims composable without pretending they are the same claim.

Terms

Law package
A bounded theorem package naming state, operations, invariants, evidence boundary, conclusion, and non-conclusions.
Local contract layer
The layer for selected projection, observation, substitution, responsibility, and dependency-inversion invariants.
Global structure layer
The layer for strict layering, decomposability, and dependency policy soundness.
State-transition layer
The layer for replay, projection, roundtrip, and compensation lawfulness.
Runtime protection layer
The layer for selected path localization, interaction protection, and bounded runtime flatness.
Distributed convergence layer
The layer for quorum, ordering, and conditional convergence under a selected failure model.
