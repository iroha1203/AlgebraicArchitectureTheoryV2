Research question

When do lawfulness, absence of required obstructions, and a zero diagnostic reading become the same statement?

AAT does not answer "whenever a tool says zero." It answers: when a selected law, required molecule family, witness family, coverage boundary, exactness boundary, and lawfulness bridge are packaged together. Zero curvature is a theorem-bound statement, not a global score.

Flatness

Flatness is the obstruction-valued reading of lawfulness. A flatness model selects architecture objects, a law universe, obstruction circuits, and an obstruction valuation. An object is flat when the selected valuation is zero.

Definition 1.1. A selected object is flat for a flatness model when the selected obstruction valuation is zero.

Proposition 1.2. Flatness implies lawfulness only when the obstruction valuation is sound for the selected law universe.

Proof idea. Soundness says that a zero valuation cannot hide a selected law failure. Without soundness, omega(A) = 0 may only mean that the selected measurement did not see a witness.

Proposition 1.3. Lawfulness implies flatness only when the obstruction valuation is complete for the selected law universe.

Proof idea. Completeness says that every selected law failure would be reflected by a nonzero obstruction valuation. Without completeness, a lawful object may not be recognizable as flat by the selected valuation.

Zero curvature

Zero curvature is the geometric name for selected obstruction-free lawfulness. A law failure can be read as a non-commuting diagram, a dependency cycle, a projection failure, a substitution failure, a state-transition mismatch, an effect replay defect, or a semantic inconsistency. Curvature is the selected valuation of that failure surface.

Definition 2.1. In the pure Atom-AAT Lean surface, ZeroCurvature law requiredMolecule is exactly NoRequiredObstructionCircuit law requiredMolecule.

Proof. Lean proves zeroCurvature_iff_noRequiredObstructionCircuit by reflexivity: the pure definition of zero curvature is the absence of required obstruction circuits.

Non-conclusion 2.2. Zero curvature is not a universal "architecture is good" claim. It is relative to the selected law, required molecules, witness coverage, exactness, and theorem package.

Theorem package

A ZeroCurvaturePackage is the complete selected package that turns no required obstruction circuits into lawfulness inside the same Atom-generated core. It carries the core, law, required molecule family, circuit-on-core evidence, lawfulness bridge, zero-curvature proof, coverage boundary, exactness boundary, theorem-package boundary, and non-conclusions.

Proposition 3.1. A zero-curvature package exposes no required obstruction circuit directly.

Proof. ZeroCurvaturePackage.noRequiredObstructionCircuit applies zeroCurvature_iff_noRequiredObstructionCircuit to the package's zeroCurvature field.

Proposition 3.2. A zero-curvature package gives lawfulness over the selected required molecule family.

Proof. ZeroCurvaturePackage.lawfulWithinMoleculeConfiguration uses LawfulnessBridge.lawful_iff_no_obstructionCircuit and the package's no-required-circuit theorem. The bridge is essential: it supplies the soundness/completeness connection between bad molecules and obstruction circuits.

Lawfulness inside a core

Zero curvature is not detached from the architecture object. In Lean, the package proves lawfulness inside the same selected AATCore that supplied the law, required molecules, and circuit evidence.

Proposition 4.1. Zero curvature gives LawfulWithinAATCore for the selected core, law, and required molecule family.

Proof. ZeroCurvaturePackage.lawfulWithinAATCore constructs a LawfulWithinAATCore record from the package's lawOnCore, requiredMoleculesOnCore, and the derived lawfulWithinMoleculeConfiguration.

Proposition 4.2. Required molecules in a zero-curvature package still contain primitive atoms from the root atom axiom system.

Proof. ZeroCurvaturePackage.atom_of_requiredMolecule uses the core's atom_of_selected_molecule theorem. The zero-curvature package never replaces the Atom root with a score or report.

Static, runtime, semantic readings

AAT distinguishes the reading in which flatness is claimed. Static flatness concerns dependency, import, module, type, or call relations. Runtime flatness concerns processes, messages, retries, timeouts, locks, and resource interactions. Semantic flatness concerns contracts, meanings, domain invariants, and effect interpretations.

Static flatness
No selected static obstruction remains, such as a cycle, hidden coupling, or forbidden dependency direction.
Runtime flatness
No selected runtime obstruction remains, such as unsafe retry, missing timeout, non-localized failure, or transaction mismatch.
Semantic flatness
No selected semantic obstruction remains, such as contract mismatch, meaning-changing substitution, or non-commuting domain diagram.

Non-conclusion 5.1. Static zero curvature does not imply runtime or semantic zero curvature unless a theorem package connects those readings.

Counterexamples and non-conclusions

Counterexample 7.1. A static dependency graph can be acyclic while a retry effect violates payment semantics. Static zero curvature does not discharge runtime-effect lawfulness.

Counterexample 7.2. A tool report can show zero selected warnings while the selected witness family is incomplete. The zero report is not a zero-curvature theorem package.

Counterexample 7.3. A zero-curvature package for one law does not prove flatness for another law. A substitution law package can pass while a state-transition law package fails.

Non-conclusion 7.4. Zero curvature does not imply empirical safety, incident absence, forecast correctness, global semantic completeness, or extractor completeness unless those claims are explicitly part of a larger theorem package.

Chapter summary

Flatness is the zero reading of selected obstruction valuation. Zero curvature is the theorem-package version of that reading: no required law-relative obstruction circuit remains in the selected molecule boundary, and the bridge assumptions turn that absence into selected lawfulness.

The boundary matters. Zero curvature is relative to a selected core, law, required molecule family, coverage boundary, exactness boundary, and theorem-package boundary. A zero score or clean report is evidence only after those conditions are made explicit.

Terms

Flatness
The selected obstruction valuation is zero.
ZeroCurvature
In pure Atom-AAT, no required obstruction circuit remains for the selected law and required molecule family.
ZeroCurvaturePackage
The theorem package carrying core, law, required molecules, bridge, zero-curvature proof, coverage, exactness, and non-conclusions.
LawfulWithinAATCore
Lawfulness over required molecules inside the same selected Atom-generated core.
Coverage boundary
The selected witness and molecule space covered by the theorem package.
Exactness boundary
The reason the selected zero reading corresponds to the intended law failure surface.
