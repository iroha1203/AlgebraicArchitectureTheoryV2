Architecture as bounded algebra

Algebraic Architecture Theory

Algebraic Architecture Theory reads a selected architecture object through operations, invariant families, obstruction witnesses, signature axes, and theorem boundaries.

AAT turns design judgment into local questions: what object, what operation, what invariant, what witness.

A one-way bridge from AAT to SFT

AAT / SFT Interface

The interface fixes dependency direction: AAT provides local algebra, while SFT uses selected AAT concepts as projections, observables, premises, and claim boundaries.

The interface is a guardrail: use AAT inside SFT, but do not promote local facts into global forecasts.

Software evolution as bounded computation

Software Field Theory

Software Field Theory models selected slices of software evolution through field state, operation support, policy, forecast cones, consequence envelopes, and feedback updates.

SFT asks how present artifacts shape the changes that become easier, harder, safer, or more costly next.

Observation tooling with explicit boundaries

ArchSig

ArchSig maps repository artifacts into measurable signature axes, witness candidates, theorem precondition reports, workflow artifacts, and explicit non-conclusions.

ArchSig keeps observation useful by keeping non-conclusions visible.
